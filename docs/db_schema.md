# Database Schema Design

WellNest uses **MongoDB** as its primary datastore, mapped via **Mongoose** schemas in TypeScript.

---

## 🧑‍🤝‍🧑 User Entities

### 1. Patient Schema (`Patient` Model)
Represents registered patient accounts.

| Field | Type | Required | Default / Description |
|---|---|---|---|
| `email` | String | Yes | Unique index, user login identifier |
| `name` | String | Yes | Patient's full name |
| `password` | String | Yes | Hashed password |
| `verificationToken` | String | No | Token used for email verification |
| `age` | Number | No | Patient age |
| `phone` | String | No | Contact phone number |
| `address` | String | No | Patient address |
| `gender` | String | No | Patient gender |
| `profileUrl` | String | No | URL to uploaded avatar image |
| `googleCalendar` | Object | Yes | Google calendar synchronization status |
| `googleCalendar.connected` | Boolean | Yes | `false` |
| `googleCalendar.refreshToken`| String | No | Persistent OAuth refresh token |
| `googleCalendar.accessToken` | String | No | Temporary OAuth access token |
| `googleCalendar.tokenExpiry`  | Date | No | Access token expiration timestamp |

---

### 2. Doctor Schema (`Doctor` Model)
Represents practicing doctors.

| Field | Type | Required | Default / Description |
|---|---|---|---|
| `adminId` | ObjectId | No | Reference to an Admin account |
| `name` | String | Yes | Doctor's full name |
| `email` | String | Yes | Unique index, doctor login identifier |
| `password` | String | Yes | Hashed password |
| `verificationToken` | String | No | Token used for email verification |
| `specialisation` | String | No | Specialty area (e.g. Cardiologist) |
| `workingHours` | Object | No | Clinic hours definition |
| `workingHours.start` | String | No | e.g., `"09:00"` |
| `workingHours.end` | String | No | e.g., `"17:00"` |
| `slotDuration` | Number | No | Duration of each appointment slot in minutes |
| `leaveDays` | Array[Date] | Yes | `[]` - Dates on which doctor is marked on leave |
| `phone` | String | No | Clinic/Doctor phone number |
| `address` | String | No | Clinic clinic address |
| `experience` | String | No | Experience summary |
| `profileUrl` | String | No | URL to avatar image |
| `qualifications` | String | No | Degree credentials (e.g. MBBS, MD) |
| `googleCalendar` | Object | Yes | Same structure as Patient `googleCalendar` |

---

### 3. Admin Schema (`Admin` Model)
Represents administrative operators who manage appointments.

| Field | Type | Required | Default / Description |
|---|---|---|---|
| `email` | String | Yes | Unique index, admin login identifier |
| `password` | String | Yes | Hashed password |
| `googleCalendar` | Object | Yes | Same structure as Patient `googleCalendar` |

---

## 📅 Scheduling Entities

### 4. Appointment Schema (`Appointment` Model)
Tracks scheduled consultations between patients and doctors.

| Field | Type | Required | Default / Description |
|---|---|---|---|
| `patientId` | ObjectId | Yes | Reference to Patient model |
| `doctorId` | ObjectId | Yes | Reference to Doctor model |
| `scheduledTime` | Date | Yes | Exact date & start time of appointment |
| `type` | String | No | `"In-Person Consultation"` |
| `status` | String | Yes | Enum: `['booked', 'rescheduled', 'cancelled', 'completed']` (default: `'booked'`) |
| `googleEventId` | String | No | Event ID returned by Google Calendar API |

---

## 🤖 Clinical & AI Entities

### 5. Symptom Schema (`Symptom` Model)
Contains raw symptoms submitted by the patient during booking.

| Field | Type | Required | Default / Description |
|---|---|---|---|
| `appointmentId` | ObjectId | Yes | Reference to Appointment model |
| `symptomText` | String | Yes | Raw symptom details / description |

---

### 6. Summary Schema (`Summary` Model)
Stores AI-generated clinical pre-visit questions/urgency classifications, and post-visit treatment plans.

| Field | Type | Required | Default / Description |
|---|---|---|---|
| `appointmentId` | ObjectId | Yes | Reference to Appointment model |
| `urgencyLevel` | String | Yes | Enum: `['low', 'medium', 'high', 'critical']` |
| `chiefComplaint` | String | Yes | Short, automated summary of main symptoms |
| `suggestedQuestions` | Array[String]| Yes | Three custom clinical follow-up questions generated by Gemini |
| `clinicalNotes` | String | No | Doctor-submitted medical notes / diagnosis |
| `prescription` | String | No | Doctor-submitted prescription details |
| `postVisitSummary` | String | No | Patient-friendly plain-language summary of clinical notes |
